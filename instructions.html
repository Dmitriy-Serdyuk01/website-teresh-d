Instructions for Modifying the Template

File Structure Overview

- index.html: The main HTML file for the homepage.
- style.css: Contains custom styles used across the website.
- normalize.css: Resets and normalizes default browser styles, also contains custom settings like colors, fonts, and background image for the body.
- main.js: JavaScript functions used for interactivity.
- images/logo/: Contains the website's logo.
- images/icon/: Contains various icons used on the website.
- images/img/: Contains general images used throughout the site.

Modifying Text Content

Open index.html in any code editor. Locate the sections using <h1>, <p>, <a>, and other text elements. Modify the text inside these tags to customize the content.

Changing Images

To update images on the site, follow these steps:

  • Logo: Replace the logo in the images/logo/ folder. Ensure that the new logo has the same name as the original file, or update the image path in the index.html or style.css file where the logo is referenced.
  • Icons: Update icons in the images/icon/ folder. Ensure the new icons follow the same naming convention or adjust the paths in the HTML or CSS files if necessary.
  • Other Images: General images for the website are stored in images/img/. Replace them as needed, ensuring the file names match, or update the paths where these images are referenced in the HTML files.

Editing Styles

To change the appearance of elements (colors, fonts, background image, etc.), open normalize.css and style.css. In normalize.css, you will find default styles like body with the background image path, fonts, and colors. Modify these values to suit your design. In style.css, you can further customize specific page elements, adding or overriding styles.

Changing Background Image

To update the background image, open normalize.css, locate the body selector, and change the background-image path to the new image you want to use.

JavaScript Customization

If you need to change interactive features, edit the corresponding functions in main.js. Make sure to test changes in multiple browsers.
